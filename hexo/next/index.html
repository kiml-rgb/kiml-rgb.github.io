next配置中遇到的问题

发表于 2022-07-20 更新于 2022-07-21 分类于 hexo

前言

受不了了，感觉自从写这个博客开始，大部分的时间就是在配置hexo和next主题，各种莫名奇妙的错误操作，所以打算自己写一篇配置记录。

官方配置网站

https://theme-next.js.org/
https://hexo-next.readthedocs.io/zh_CN/latest/

样式修改

更换代码高亮

在官方提供的样式网站查看了几种样式，发现原本默认的highlight.js渲染java语言基本上就是不怎么变色的一个状态，在hexo主题下显示变得尤为单调，于是想用官方提供的prism引擎下的代码样式，在hexo文件下的_comfig.yml文件中，可以看到默认启用的是highlight.js

highlight:
  enable: true
  line_number: true
  auto_detect: false
  tab_replace: ''
  wrap: true
  hljs: false
prismjs:
  enable: false
  preprocess: true
  line_number: true
  tab_replace: ''

默认情况下禁用 Prismjs。您应该在启用 prismjs 之前设置highlight.enable为:false。但是就是这一步，明明已经设置为prismjs，但是在next的主题下设置完代码高亮的样式之后，发生了如下错误

FATAL {
  err: Error: ENOENT: no such file or directory, open 'D:\Blog\node_modules\highlight.js/styles/prism-coy.css'

后来发现问题在于next主题下的_comfig.yml配置文件，我做的就是把theme下的两个都设置成了default，之后就可以使用prism代码高亮了，至于为什么我也不知道

codeblock:
  # Code Highlight theme
  # All available themes: https://theme-next.js.org/highlight/
  theme:
    light: default
    dark: default
  prism:
    light: prism-coy
    dark: prism-coldark-dark
  # Add copy button on codeblock
  copy_button:
    enable: true
    # Available values: default | flat | mac
    style: mac

注意：使用相关的代码高亮需要下载
highlight下载为

$ hexo install highlightjs

prism下载为：

$ hexo install prismjs

在官方提供的代码块高亮演示界面中有一个样式是prism-coy-without-shadows，但是直接在light处添加这个主题是找不到的，我们可以直接去对应的目录\Blog\node_modules\prismjs\themes下面，可以看到演示界面所给出的Prism addtional themes分类下的文件在这里都是找不到的，但是我想要的又是prism-coy-without-shadows样式，没办法，修改prism-coy.css源文件：

pre[class*="language-"]:before,
pre[class*="language-"]:after {
	content: '';
	display: none;

将这一段的display处修改为none就能得到我想要的结果，至于别的我没试过，也不知道怎么得到。

其他问题

三种布局

这里倒是没有错误，但是查阅官方文档，突然发现hexo指定了3种布局post(文章)、page(页面)和draft(草稿)，完整的添加布局的指令为

$ hexo new [layout] <title>

这里比较常用的就是草稿文件，可以手动进行操作，也可以使用命令hexo publish实现：将草稿箱_drafts的文件发布到_posts文件夹

$ hexo publish draft Draft-Test
INFO  Validating config
INFO  Published: ~/blogs/source/_posts/2021-03-10-Draft-Test.md

完成发布后，会添加相应的日期信息（基于post.md模板）：

---
title: Draft Test
date: 2021-03-10 22:05:24
tags:
---

由于默认情况下，服务器不会读取草稿箱内的文件，所以对我来说是一个比较方便的功能。
