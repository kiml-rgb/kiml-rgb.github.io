JAVA 8 四大函数型接口

发表于 2022-07-09 更新于 2022-07-21 分类于 Java ， Java 8 新特性

前言

学lambda表达式，针对这一部分进行了拓展。工作中会经常用到这四大函数型接口。

函数式接口

函数式接口(Functional Interface)就是一个有且仅有一个抽象方法，但是可以有多个非抽象方法的接口。

常用注解

lambda表达式的使用前提是接口为函数式接口，对于函数式接口，java提供了一个注释@FunctionalInterface，用于检测当前接口是否只含有一个抽象方法。
补充
今天在查看Comparator接口源码的时候，发现这个接口使用@FunctionalInterface注释修饰，但是类中有两个抽象方法。

查到的解释如下：

接口默认继承java.lang.Object，所以如果接口显示声明覆盖了Object中方法，那么也不算抽象方法

但是还是不理解此接口覆盖equals方法的用意，也没有找到相应的例子。在API文档中可以看到

Note that it is always safe not to override Object.equals(Object). However, overriding this method may, in some cases, improve performance by allowing programs to determine that two distinct comparators impose the same order.

可能后续遇到了，才会有所理解。

四大函数式接口

函数式接口 参数类型 返回值类型 描述
Consumer T 无 消费型
Supplier 无 T 供给型
Function T R 函数型
Predicate T boolean 断言型
消费型接口

接受一个输入参数并且无返回值

@FunctionalInterface
public interface Consumer<T> {
    void accept(T t);

    default Consumer<T> andThen(Consumer<? super T> after) {
        Objects.requireNonNull(after);
        return (T t) -> { accept(t); after.accept(t); };
    }
}
基本格式
Consumer<T>
方法调用
void accept(T t);
示例
public class Test {
    public static void main(String[] args) {
        handle(5, (x) -> System.out.println(x + 2));
    }

    public static void handle(int value, Consumer<Integer> consumer) {
        consumer.accept(value);
    }
}
用法

因为没有出参，所以常用于打印、发送短信等消费动作。

供给型接口

无输入参数，返回一个结果T。

@FunctionalInterface
public interface Supplier<T> {
    T get();
}
基本格式
Supplier<T>
方法调用
T get();
示例
public class Test {
    public static void main(String[] args) {
        System.out.println(getObject(() -> "Supplier"));
    }

    public static String getObject(Supplier<String> supplier) {
        return supplier.get();
    }
}
用法

常用于符合条件时调用获取结果；运行结果提前定义，但不运行

函数型接口

接受一个输入参数T，返回一个结果R。

@FunctionalInterface
public interface Function<T, R> {

    R apply(T t);

    default <V> Function<V, R> compose(Function<? super V, ? extends T> before) {
        Objects.requireNonNull(before);
        return (V v) -> apply(before.apply(v));
    }

    default <V> Function<T, V> andThen(Function<? super R, ? extends V> after) {
        Objects.requireNonNull(after);
        return (T t) -> after.apply(apply(t));
    }

    static <T> Function<T, T> identity() {
        return t -> t;
    }
}
基本格式
Function<T,R>
方法调用
R apply(T t);
示例
public class Test {
    public static void main(String[] args) {
        int i = plusTwo(5, x -> x + 2);
        System.out.println(i);

        int before = before(5, x -> x + 2, x -> x * 2);
        System.out.println(before);

        int after = after(5, x -> x + 2, x -> x * 2);
        System.out.println(after);
    }

    public static Integer plusTwo(int origen, Function<Integer, Integer> function) {
        return function.apply(origen);
    }

    public static Integer before(int value, Function<Integer, Integer> function1, Function<Integer, Integer> function2) {
        // value作为function2的参数，返回一个结果，该结果作为function1的参数，返回一个最终结果
        return function1.compose(function2).apply(value);
    }

    public static Integer after(int value, Function<Integer, Integer> function1, Function<Integer, Integer> function2) {
        // value作为function1的参数，返回一个结果，该结果作为function2的参数，返回一个最终结果
        return function1.andThen(function2).apply(value);
    }
}
断言型接口

接受一个输入参数T，返回一个布尔值结果。

@FunctionalInterface
public interface Predicate<T> {

    boolean test(T t);

    default Predicate<T> and(Predicate<? super T> other) {
        Objects.requireNonNull(other);
        return (t) -> test(t) && other.test(t);
    }

    default Predicate<T> negate() {
        return (t) -> !test(t);
    }

    default Predicate<T> or(Predicate<? super T> other) {
        Objects.requireNonNull(other);
        return (t) -> test(t) || other.test(t);
    }

    static <T> Predicate<T> isEqual(Object targetRef) {
        return (null == targetRef)
                ? Objects::isNull
                : object -> targetRef.equals(object);
    }

    @SuppressWarnings("unchecked")
    static <T> Predicate<T> not(Predicate<? super T> target) {
        Objects.requireNonNull(target);
        return (Predicate<T>)target.negate();
    }
}
基本格式
Predicate<T>
方法调用
boolean test(T t);
示例
public class Test {
    public static void main(String[] args) {
        List<Integer> list = Arrays.asList(1, 2, 3, 4, 5, 6, 7, 8, 9);
        // 输出所有数据
        eval(list, x -> true);
        System.out.println();

        // 输出所有偶数
        eval(list, x -> (x & 1) != 1);
        System.out.println();

        // 输出所有大于3的数
        eval(list, x -> x > 3);
        System.out.println();

        // 判断输入是否不为空并且长度小于3
        boolean valid = valid("许仙", x -> !x.isEmpty() && x.length() <= 3);
        System.out.println(valid);
    }

    public static boolean judge(Integer input, Predicate<Integer> predicate) {
        return predicate.test(input);
    }

    public static void eval(List<Integer> list, Predicate<Integer> predicate) {
        for (Integer integer : list) {
            if (predicate.test(integer)) {
                System.out.print(integer + " ");
            }
        }
    }

    public static boolean valid(String name, Predicate<String> predicate) {
        return predicate.test(name);
    }
}

常用于条件判断
