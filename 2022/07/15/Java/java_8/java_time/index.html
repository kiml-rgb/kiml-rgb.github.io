JAVA 8 日期时间API

发表于 2022-07-15 更新于 2022-07-21 分类于 Java ， Java 8 新特性

前言

对于JAVA 8中新引入的日期时间API只是简单的提了一嘴，但是又说很好用，建议大家去了解一下（是不是不发火就。。。）
于是就自己查阅资料并试了几个例子。目前学习的java.util.Date，是在JDK 1.0的时候引入的，目前已知这个API存在诸多问题；在JDK 1.1的时候引入了功能更强大的java.util.Calendar，但是Date存在的部分问题还是没有解决。在JDK 1.8的时候，Java引入了新的API，这才真正修改了过去的缺陷，且更为好用。本篇就详细介绍一下JDK 1.8的日期和时间API。

java.time包结构

新版java.time下面分为5个部分，大部分情况下用到基础包和format包就行

java.time – 包含值对象的基础包
java.time.chrono – 提供对不同的日历系统的访问
java.time.format – 格式化和解析时间和日期
java.time.temporal – 包括底层框架和扩展特性
java.time.zone – 包含时区支持的类

基础包的使用

java.time.LocalDateTime

java.time.LocalDateTime包括年月日时分秒，java.time.LocalDate包括年月日，java.time.LocalTime包括时分秒，在源码中可以看到java.time.LocalDateTime就是在类中创建了两个LocalDate和LocalTime的final对象，并对其中的方法进行调用，因此这里只介绍java.time.LocalDateTime中的方法。

  • 从日期和时间中获取单独部分

import java.time.LocalDateTime;
public class LocalDateTimeTest {
    public static void main(String[] args) {
        LocalDateTime now = LocalDateTime.now();
        // 获取当前时间 格式为：2022-07-16T21:20:50.325202900
        System.out.println(now.toString());
        // 获取当前年份 返回int值
        System.out.println(now.getYear());
        // 获取当前月份 返回enum值
        System.out.println(now.getMonth());
        // 获取当前月份 返回int值
        System.out.println(now.getMonthValue());
        // 获取当前日期是所在年份的第几天 返回int值
        System.out.println(now.getDayOfYear());
        // 获取当前日期是所在月份的第几天 返回int值
        System.out.println(now.getDayOfMonth());
        // 获取星期 返回enum值
        System.out.println(now.getDayOfWeek());
        // 获取小时 返回int值 24小时制
        System.out.println(now.getHour());
        // 获取分钟 返回int值
        System.out.println(now.getMinute());
        // 获取秒数 返回int值
        System.out.println(now.getSecond());
        // 获取纳秒
        System.out.println(now.getNano());
    }
}

  • 简单增删改

// 自定义时间
LocalDateTime time = LocalDateTime.of(2000, 2, 29, 21, 21, 21);
System.out.println(time);

// 使用plus方法增加年份 同理增加年月日时分秒毫秒星期都行
// 此方法返回一个新的对象，原有对象值不能修改
LocalDateTime plusYears = time.plusYears(22);
System.out.println(time);
System.out.println(plusYears);

// 使用minus方法减少年份
LocalDateTime minusYears = time.minusYears(1);
System.out.println(minusYears);

// 使用with方法设置年份
LocalDateTime withYear = time.withYear(2018);
System.out.println(withYear);

// 判断当前年份是否为闰年
LocalDate date = LocalDate.now();
System.out.println(date.isLeapYear());

需要注意的是isLeapYear()这个方法是LocalDate类所特有的

  • 复杂的日期操作比较复杂的日期操作，比如将时间调到下一个工作日，或者是下个月的最后一天，这时候我们可以使用with()方法的另一个重载方法，它接收一个TemporalAdjuster参数，可以使我们更加灵活的调整日期。
    底层。。。暂时不想研究。总之，根据jdk提供的注解只需要知道可以使用，并且推荐使用TemporalAdjusters中提供的方法就行。TemporalAdjusters类在java.time.temporal中。因此关于with()传入TemporalAdjuster参数的方法演示标记在了目录java.time.temporal.TemporalAdjusters之下。

    There are two equivalent ways of using a TemporalAdjuster. The first is to invoke the method on this interface directly. The second is to use Temporal.with(TemporalAdjuster):
    // these two lines are equivalent, but the second approach is recommended
    temporal = thisAdjuster.adjustInto(temporal);
    temporal = temporal.with(thisAdjuster);
    It is recommended to use the second approach, with(TemporalAdjuster), as it is a lot clearer to read in code.

java.time.Instant

  • 常用方法

import java.time.Instant;
public class InstantTest {
    public static void main(String[] args) {
        Instant now = Instant.now();

        // 获取当前时间毫秒值 返回long值
        System.out.println(now.toEpochMilli());
        // 获取当前时间的秒值 返回long值
        System.out.println(now.getEpochSecond());

        // 判断当前时间是否在参数中的时间之后
        System.out.println(now.isAfter(now));
        // 判断当前时间是否在参数中的时间之前
        System.out.println(now.isBefore(now));
        // 当前时间与参数中的时间进行对比，在参数之前，相同，之后分别为（-1, 0, 1）
        System.out.println(now.compareTo(now));

        // 用一个时间戳去创建一个Instance实例，用法和new Date（时间戳）创建一个Date实例是一样的
        Instant.ofEpochMilli(System.currentTimeMillis());
    }
}
java.time.Instant和java.time.LocalDateTime互转
import java.time.Instant;
import java.time.LocalDateTime;
import java.time.ZoneId;
public class Instant2LocalDateTime {
    public static void main(String[] args) {
        Instant instant = Instant.ofEpochMilli(System.currentTimeMillis());
        // 时区设置为系统默认时区
        ZoneId zoneId = ZoneId.systemDefault();
        LocalDateTime instant2LocalDateTime = LocalDateTime.ofInstant(instant, zoneId);
        
        LocalDateTime now = LocalDateTime.now();
        Instant localDateTime2Instant = now.atZone(ZoneId.systemDefault()).toInstant();
    }
}
java.time.Duration
import java.time.Duration;
import java.time.LocalDateTime;
public class DurationTest {
    public static void main(String[] args) {
        LocalDateTime startTime = LocalDateTime.of(2000, 2, 29, 0, 0);
        LocalDateTime endTime = LocalDateTime.now();

        Duration between = Duration.between(startTime, endTime);
        System.out.println(between);
        // 计算同类型日期的时间差，也可以to分秒...
        // 这两个方法在源码里看起来好像没区别 toDaysPart()是在jdk1.9提供的
        System.out.println(between.toDays());
        System.out.println(between.toDaysPart());
    }
}

java.time.chrono包介绍

除默认ISO之外的日历系统的通用API。
主API基于ISO-8601中定义的日历系统（即日常用的公历）。但是，还有其他日历系统，此软件包为它们提供基本支持。java.time.chrono包中提供了备用日历。
支持的日历系统包括：

  • Hijrah calendar

  • Japanese calendar

  • Minguo calendar

  • Thai Buddhist calendar

现阶段应该用不到，了解一下就行

java.time.format包介绍

提供打印和解析日期和时间的类。
打印和解析基于DateTimeFormatter类。实例通常从DateTimeFormatter获得，但是如果需要更多功率则可以使用DateTimeFormatterBuilder。

java.time.format.DateTimeFormatter

java.time.zone包介绍

支持时区及其规则。
夏令时和时区是各国政府用来改变当地时间的概念。此软件包为时区，其规则以及由夏令时引起的本地时间线产生的间隙和重叠提供支持。
定义了时区转换的各种方法。

java.time.temporal包介绍

使用字段和单位以及日期时间调整器访问日期和时间。
该软件包扩展了基础软件包，为更强大的用例提供了额外的功能。支持包括：

  • 日期时间单位，例如年，月，日和小时

  • 日期时间字段，例如月份，星期几或小时

  • 日期时间调整功能

  • 周的不同定义

Java8中的日期时间类都是实现了temporal包中的时间单位（Temporal）、时间调节器（TemporalAdjuster）和各种属性的接口，所以在后面的日期的操作方法中都是以最基本的时间单位和各种属性为参数的。

java.time.temporal.TemporalAdjusters

  • 复杂的日期操作使用格式如下：

    LocalDateTime now = LocalDateTime.now();

// 返回当前月份的第一天
LocalDateTime firstDayOfMonth = now.with(TemporalAdjusters.firstDayOfMonth());
System.out.println(firstDayOfMonth);

    下面列出时间调节器TemporalAdjuster提供的一些方法

    方法名 描述
    dayOfWeekInMonth 返回同一个月中每周的第几天
    firstDayOfMonth 返回当月的第一天
    firstDayOfNextMonth 返回下月的第一天
    firstDayOfNextYear 返回下一年的第一天
    firstDayOfYear 返回本年的第一天
    firstInMonth 返回同一个月中第一个星期几
    lastDayOfMonth 返回当月的最后一天
    lastDayOfNextMonth 返回下月的最后一天
    lastDayOfNextYear 返回下一年的最后一天
    lastDayOfYear 返回本年的最后一天
    lastInMonth 返回同一个月中最后一个星期几
    next / previous 返回后一个/前一个给定的星期几
    nextOrSame / previousOrSame 返回后一个/前一个给定的星期几，如果这个值满足条件，直接返回

新旧API比较

旧版API存在的问题

  • 非线程安全：java.util.Date是非线程安全的，所有的日期类都是可变的，这是Java日期类最大的问题之一。

  • 设计很差：Java的日期/时间类的定义并不一致，在java.util和java.sql的包中都有日期类，此外用于格式化和解析的类在java.text包中定义。java.util.Date同时包含日期和时间，而java.sql.Date仅包含日期，将其纳入java.sql包并不合理。另外这两个类都有相同的名字，这本身就是一个非常糟糕的设计。

  • 时区处理很麻烦：日期类并不提供国际化，没有时区支持，因此Java引入了java.util.Calendar和java.util.TimeZone类，但他们同样存在上述所有的问题。

  • 编码问题：java.util.Calendar中的月份从0开始，从中获取月份需要加一才能表示当前月份；星期的计算从星期日开始，这些都使得它的使用变得不便。

新版API优势

  • 它涵盖了所有处理日期，时间，日期/时间，时区，时刻（instants），过程（during）与时钟（clock）的操作。

  • 所有的类都是不可变的、线程安全的。

  • 新时间API类都实现了一系列方法用以完成通用的任务，如：加、减、格式化、解析、从日期/时间中提取单独部分，等等

  • time包里面的类实例如果用了上面的方法而被修改了,那么会返回一个新的实例过来,而不像Calendar那样可以在同一个实例进行不同的修改,体现了不可变

在jdk8中可以使用Instant代替Date，LocalDateTime代替Calendar，DateTimeFormatter代替SimpleDateFormat。

java.util.Date或java.util.Calendar到新库类的转换
import java.time.Instant;
import java.util.Calendar;
import java.util.Date;
public class Test {
    public static void main(String[] args) {
        // 旧版Date转新版Instant
        Date date = new Date();
        Instant date2Instant = date.toInstant();

        // 新版Instant转旧版Date
        Instant instant = Instant.now();
        Date instant2Date = Date.from(instant);

        // 旧版Calendar转成新版Instant
        Calendar calendar = Calendar.getInstance();
        Instant calendar2Instant = calendar.toInstant();
    }
}
