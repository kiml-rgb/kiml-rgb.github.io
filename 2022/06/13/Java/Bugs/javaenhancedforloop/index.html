java数组遍历--foreach

发表于 2022-06-13 更新于 2022-07-13 分类于 Java ， Bugs

前言

在练习数组的使用时，想用foreach循环给数组赋值，但得到了一个空数组。

bug出处

原示例代码：

import java.util.Random;
public class Test {
    public static void main(String[] args) {
        Random random = new Random();
        int arr[] = new int[10];
        String s = "[";

        for (int i = 0 ; i < arr.length; i++) {
            arr[i] = random.nextInt(889-22) + 22;//[0, 889-22) + 22
        }

        System.out.println("打印数组元素：");
        for (int i : arr) {
            s += i + ", ";
        }
        s = s.substring(0, s.length()-2) + "]";
        System.out.println(s);

        System.out.println("打印数组中是3的整倍数的元素：");
        s = "[";
        for (int i : arr) {
            if (i % 3 == 0) {
                s += i + ", ";
            }
        }
        s = s.substring(0, s.length()-2) + "]";
        System.out.println(s);
    }
}

此段代码给数组元素赋值时，使用foreach循环，赋值失败。通过后续查资料想进一步完善其用法，发现foreach循环不是完全的不能改变原有数组的值。

四种情况

首先foreach的格式如下：

For (类型 变量名:集合) { 语句块; }

此处的 类型 变量名 指的是使用第三方变量复制了迭代指向的数据。这里我简单的理解为一个temp。

基本数据类型数组（int）
int[] list1 = new int[10];
// 迭代，循环依次指向数组中元素，用第三方变量i记录
// 没有对原有数据作出改变
for (int i : list1) {
    i = 10;
}
System.out.println(Arrays.toString(list1));

与原有的例子一样，此处遍历得到的结果如下：

既然foreach只是用第三方变量记录，记录基本数据类型不存在赋值过程，自然不会对原数组作出改变。如此，记录引用数据类型，记录的是地址，照理说应该能对原数组作出改变了。

引用数据类型数组（String）
List<String> list2 = new ArrayList<>();
for (int i = 0; i < 10; i++) {
    list2.add("s");
}
// 迭代，循环依次指向数组中元素，用第三方变量s记录
// 引用数据类型，改变原有地址值中的数据
// 
for (String s : list2) {
    s = "l";
}
System.out.println(list2.toString());

此处遍历得到的结果如下：

对原数组依然没有作出改变，理由是String被final修饰，s = "l"只是在内存中new了一个对象，使String指向新的对象，但是此处s为第三变量，第三变量的引用改变，对于原list中记录的值不会有任何改变。既然如此，记录了地址的二维数组，没有被final修饰，应该就可以改变第二层循环内的的数据了。

引用数据类型 （二维数组（int[][]））
int[] a1 = {1, 2, 3};
int[] a2 = {4, 5, 6};
int[] a3 = {7, 8, 9};
int[][] arr = {a1, a2, a3};

// 迭代，循环依次指向数组中元素，用第三方变量s记录
// 引用数据类型，改变原有地址值中的数据
// arr没有被final修饰，此处就是在源地址处改变，改变成功
for (int[] ints : arr) {
    ints[1] = 2;
}
for (int[] ints : arr) {
    System.out.println(Arrays.toString(ints));
}

此处遍历得到的结果如下：

可以看到第二层循环内的改变成功。如此推断，如果数组中的元素是没有被final修饰的引用数据类型，应该就是可以通过foreach改变。可以自建一个JavaBean格式的Num类测试一下。

引用数据类型（普通类）
List<Num> list3 = new ArrayList<>();
for (int i = 0; i < 10; i++) {
    list3.add(new Num("s"));
}
// 迭代，循环依次指向数组中元素，用第三方变量s记录
// 引用数据类型，记录为地址，改变原有地址值中的数据
// 改变成功
for (Num num : list3) {
    num.setNum("l");
}
System.out.println(list3);

此处遍历得到的结果如下：

可以看到，改变成功。当然如果只是final修饰类，没有修饰属性，属性也是可以进行修改的。

总结:数组中元素是引用数据类型，并且该数据类型不是jdk提供的被final修饰的数据类型（例如String, Integer…），那么就可以通过foreach修改原数组中存储的数据。此处也有点像方法中的传参。
