找不到或无法加载主类 java.lang.ClassNotFoundException

发表于 2022-06-10 更新于 2022-07-22 分类于 Java ， Bugs

前言

很尴尬，找不到工作来报了个班，结果一开始用nodepad++写代码编译就出问题，为什么之前没有我也不知道。。。

可以编译成功，PATH环境变量配置正确，代码无问题时出现以下错误。

引起原因

1、CLASSPATH环境变量配置问题
java指令默认在寻找class文件时是根据CLASSPATH环境变量指定的目录寻找的。环境变量配置应该如下：

.;%JAVE_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar

其中“.”表示当前文件夹，原先配置时，“.”后忘记添加“;”导致出现错误java.lang.ClassNotFoundException

拓展

注意： 在定义CLASSPATH环境变量时，如果没有定义环境变量CLASSPATH，java启动jvm后，会在当前目录下查找要运行的类文件；如果指定了CLASSPATH，那么会在指定的目录下查找要运行的类文件。

CLASSPATH是什么？它的作用是什么？
它是javac编译器的一个环境变量。它的作用与import、package关键字有关。当你写下improt java.util.*时，编译器面对import关键字时，就知道你要引入java.util这个package中的类；但是编译器如何知道你把这个package放在哪里了呢？所以你首先得告诉编译器这个package的所在位置。如果java.util这个package在c:/jdk/目录下，把c:/jdk/这个路径设置到CLASSPATH中，当编译器面对import java.util.*这个语句时，它先会查找CLASSPATH所指定的目录，并检视子目录java/util是否存在，然后找出名称吻合的已编译文件（.class文件）。
JDK 5.0开始，默认在当前工作目录即上述设置中寻找Java程序。因此，在安装之后，即使完全没有设定CLASSPATH，你仍然能够编译基本的JAVA程序，并且加以执行。总之，设置CLASSPATH的目的，在于告诉Java执行环境，在哪些目录下可以找到所要执行的Java程序（.class文件）
